# hthadmin-resource

## 项目简介

`hthadmin-resource` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态资源存储与分发的通用空间。

仓库中的 HTML 页面各自独立，彼此无依赖关系，可直接通过浏览器打开或通过静态托管服务访问。

## 目录说明

```
hthadmin-resource/
├── pages/          # 存放归档的 HTML 页面文件
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── README.md       # 本文件
└── .gitignore      # Git 忽略规则
```

- **pages/**：所有独立 HTML 页面均放置于此目录下，每个页面以有意义的名字命名，便于识别。
- **assets/**：存放页面所需的公共资源，如样式表、脚本、图标等。若页面需要使用独立资源，建议在页面同级目录下创建子文件夹管理。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，不依赖外部服务或数据库。每个页面包含完整的 HTML 结构与所需资源引用。

### 归档规范

- 页面文件名应简洁明了，避免特殊字符。
- 页面内部资源请使用相对路径引用，确保可移植性。
- 如需引用外部资源（如 CDN 库），请注明来源版本。

### 使用方式

1. 克隆或下载本仓库到本地。
2. 直接双击 `pages/` 下的 HTML 文件即可在浏览器中查看。
3. 也可通过 GitHub Pages 或其他静态托管服务发布。

## 维护说明

### 添加新页面

1. 在 `pages/` 目录下创建新的 HTML 文件。
2. 确保页面内部资源引用路径正确。
3. 如有公共资源需求，请放入 `assets/` 目录。
4. 提交 PR 或直接推送至主分支。

### 更新已有页面

- 直接修改对应 HTML 文件，并保持文件结构不变。
- 修改公共资源时，需注意是否影响其他页面。

### 删除页面

- 移除 `pages/` 下的对应文件，同时清理不再使用的资源文件。

## 注意事项

- 本仓库仅用于存放静态 HTML 页面，不包含后端逻辑或动态内容。
- 请勿在页面中嵌入跟踪代码、广告或其他违反 GitHub 服务条款的内容。
- 所有页面内容由贡献者自行负责，请确保不包含敏感或侵权信息。
